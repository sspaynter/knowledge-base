active · v1.2 · 2026-03-02 · by simon

How Claude Code is structured across Simon's projects — agents, skills, background rules, and their relationships. This page documents the hierarchy so it can be maintained as the system evolves.

Agent and Skill Hierarchy

Five global agents are available in every project. Background skills load silently. Job-search-specific skills activate when working in the job-app project.

Scoring Pipeline

Job alerts from Gmail flow through a two-step AI scoring pipeline on n8n. Haiku triages for track fit and red flags, then Sonnet performs detailed 8-dimension scoring.

flowchart LR A["📧 Gmail Alert"] --> B["Parse Email"] B --> C["Fetch Job Page"] C --> D{"Triage\n(Haiku)"} D -->|Pass| E["Score\n(Sonnet)"] D -->|Reject| F["Skip"] E --> G[("PostgreSQL")] G --> H["NocoDB\nView"]

Infrastructure

All services run on a QNAP NAS via Docker containers. Cloudflare Tunnel provides external access without port forwarding. Each app has staging and production containers behind Watchtower for auto-deploy.
