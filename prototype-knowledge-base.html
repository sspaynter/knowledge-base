simon-context

active skill global · v1.4 · Updated 2026-02-27 · by claude

The simon-context skill provides Claude with a complete working profile of Simon Paynter — cognitive style, communication preferences, execution framework, and product context. It loads at the start of every session via the global CLAUDE.md.

Purpose

Claude starts each session without memory of previous interactions. This skill solves the context-rebuild problem by encoding Simon's stable working preferences as a structured reference that Claude reads before doing anything else.

Without it, Claude defaults to generic behaviour: verbose explanations, corporate tone, no knowledge of how Simon thinks or what he has already built.

Coverage

  • User context — Name, location, role, primary domains
  • Cognitive style — ADHD, dyslexia, architectural thinking, execution preferences
  • Communication preferences — Plain language, no jargon, direct tone
  • Execution framework — How to help when stuck, how to sequence work
  • Technical context — Tools, platforms, current projects, infrastructure
  • Response format — Structure, heading use, step size, table preference
  • Session-end protocol — Planning doc updates, skill gap checks, docs folder updates

Change log

This skill is updated at the end of any session where preferences, tools, or context meaningfully changed. Changes are versioned in asset_versions.
