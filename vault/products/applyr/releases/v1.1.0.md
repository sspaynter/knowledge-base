# Applyr v1.1.0 — Cover Letter Prompt Architecture

**Released:** 2026-03-14
**Tag:** [v1.1.0](https://github.com/sspaynter/applyr/releases/tag/v1.1.0)
**Commits since v1.0.0:** 15

## Summary

v1.1.0 introduces a 3-layer cover letter prompt architecture (Craft, Quality, Voice) with 281 unit tests. Cover letter generation now uses positive structural constraints, anonymised examples, and fabrication guards. The prompt service was split into pure functions to fix 9 consecutive CI failures. Anti-slop quality rules and intent-based opening guidance improve output quality. CI now builds semver-tagged Docker images for version-pinned rollback.

## Changes

### Added

- 3-layer cover letter prompt architecture: Craft (structure), Quality (anti-slop), Voice (personalisation)
- Anti-slop quality rules banning intensifiers, jargon, filler, throat-clearing, hedging
- Voice profile rhythm and naturalness principles
- Positive structural constraints with anonymised example openings
- Intent-based opening guidance — first-person perspective, company detail connected to experience
- Discovery framing for opening paragraph
- 281 unit tests for prompt assembly functions (up from 75)
- Semver-tagged Docker images for version-pinned rollback
- SESSION_SECRET safety net in CI environment

### Fixed

- CI test failures caused by coverLetter.js importing db.js/config.js (9 consecutive failures resolved)
- Fabrication guard — model cannot invent domain expertise the applicant does not have
- Anonymised company names in examples to prevent verbatim copying
- Resolved conflicting prompt instructions between framework and format sections
- Opening paragraph must land on insight, not self-reference

### Changed

- Extracted pure prompt functions into `services/coverLetterPrompts.js` (service module boundary convention)
- Service module boundary rule codified in CLAUDE.md
