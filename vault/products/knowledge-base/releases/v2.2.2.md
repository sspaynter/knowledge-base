# v2.2.2 — Auth Resilience

**Released:** 2026-03-19
**Type:** Patch (bug fix)
**Scope:** #207 — re-login quick fix

## Changes

### Fixed
- Shared auth pool `max` increased from 5 to 10 connections
- Connection timeout increased from 5s to 10s
- Frontend `api.js` retries once on transient 401 when user was previously authenticated

### Added
- `services/request-retry.js` — extracted retry logic for testability
- 9 new tests (2 pool config + 7 retry behaviour)

## Context

Clicking between sections intermittently forced re-login. Root cause: `sharedAuthPool` (max 5, 5s timeout) connecting to `applyr_staging` for session deserialization was a bottleneck under concurrent requests.

This is a quick fix. The proper fix is #208 (DB consolidation) which eliminates the separate shared auth pool entirely.
