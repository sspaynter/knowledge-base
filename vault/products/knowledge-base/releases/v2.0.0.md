# Knowledge Base v2.0.0

**Released:** 2026-03-04
**Tag:** [v2.0.0](https://github.com/sspaynter/knowledge-base/releases/tag/v2.0.0)
**Production:** https://kb.ss-42.com
**Staging:** https://kb-staging.ss-42.com

## Summary

Complete rewrite of the Knowledge Platform across 6 phases. Pages served from an Obsidian-compatible vault with three-layer read fallback (vault file, content cache, database). Responsive frontend with rail app switcher, Mermaid diagrams, full-text search, dark mode, PWA, and Google OAuth SSO.

## Changes by Phase

### Phase 1 — Vault Sync + Visual Redesign
- Vault sync engine (chokidar v5, ESM lazy-load)
- Three-layer content fallback
- Mermaid diagram rendering
- Indigo visual design, Plus Jakarta Sans

### Phase 2 — Editor + Write Paths
- Split-view editor (markdown + preview)
- Mermaid click-to-edit with templates
- Page version history
- Vault-first writes

### Phase 3 — Navigation + Responsive Layout
- Two-tier rail app switcher
- Mobile drawer
- Workspace/section sidebar

### Phase 4 — PWA + Search + Export
- Service worker, installable PWA
- Full-text search (PostgreSQL tsvector)
- Diagram SVG/PNG export
- Dark mode

### Phase 5 — Migration + Features
- v1-to-v2 migration script
- Inbox, asset browser, status filtering, map toggle

### Phase 6 — Testing + CI/CD + Taxonomy
- 88 tests, 17 suites
- CI test gate on push
- 9-workspace PARA-informed vault taxonomy
- Healthcheck IPv6 fix, incremental migrations

## Rollback

```bash
# On NAS (via SSH):
DOCKER=/share/CACHEDEV2_DATA/.qpkg/container-station/usr/bin/docker

# Stop and remove current container
$DOCKER stop knowledge-base && $DOCKER rm knowledge-base

# Pull previous image (v1 had no version tag — use sha-based image if available)
# Or rebuild from the commit before v2.0.0:
#   git checkout 214705f  (last v1 commit on main)
#   Build and push manually

# Recreate with v1 config (no vault mount needed):
$DOCKER run -d --name knowledge-base \
  --restart always \
  -p 32781:3000 \
  -v /share/CACHEDEV1_DATA/Container/knowledge-base/data:/app/uploads \
  -e NODE_ENV=production \
  -e PORT=3000 \
  -e DATABASE_URL=postgresql://kb_app:eb8aa0f9f18a0e279fc05f8b836cd328750d542acb3ba2f3@10.0.3.12:5432/nocodb \
  -e SESSION_SECRET=6b2cf1028c83fdcfba7766995d75c9dac8b112ed240a4686032fbef482ba5c96 \
  -e HQ_URL=https://hq.ss-42.com \
  ghcr.io/sspaynter/knowledge-base:{previous-sha}
```
