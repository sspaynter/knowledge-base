# Knowledge Base v2.0.1

**Released:** 2026-03-04
**Tag:** [v2.0.1](https://github.com/sspaynter/knowledge-base/releases/tag/v2.0.1)
**Previous:** [v2.0.0](https://github.com/sspaynter/knowledge-base/releases/tag/v2.0.0)
**Production:** https://kb.ss-42.com
**Staging:** https://kb-staging.ss-42.com

## Summary

v2.0.1 adds the infrastructure for structured page management: frontmatter metadata parsing, gapped sort ordering, and drag-to-reorder in the sidebar. These three features form a dependency chain (#35 → #36 → #13) that gives users direct control over page order for the first time. A version label in the app rail makes it easy to identify which version is running.

## Features

### Frontmatter metadata parsing (#35)

Vault `.md` files can now include YAML frontmatter between `---` delimiters. Seven fields are mapped to database columns:

| Frontmatter field | DB column | Notes |
|---|---|---|
| `status` | `status` | published, draft, archived |
| `order` | `sort_order` | Numeric sort position |
| `parent` | `parent_id` | Slug of parent page in same section |
| `author` | `created_by` | claude, user, both |
| `title` | `title` | Overrides filename-derived title |
| `created` | `created_at` | ISO date |
| `updated` | `updated_at` | ISO date |

Round-trip: `writeVaultFile` prepends frontmatter on save. Frontmatter is stripped from `content_cache` so it does not appear in rendered pages.

New module: `services/frontmatter.js` (js-yaml dependency added).
Migration 004: adds `archived` status value.

### Gapped sort ordering (#36)

All sort_order values now use gapped numbering (10, 20, 30...) to allow future insertions without renumbering:

- Seed data: 9 workspaces (10-90), 14 sections (10-40 per workspace)
- Vault sync: reads `order` from frontmatter; falls back to `MAX(sort_order) + 10`
- Auto-created workspaces and sections: `MAX(sort_order) + 10`

### Drag-to-reorder pages (#13)

Native HTML5 drag-and-drop in the sidebar. Pages within a section can be reordered by dragging. Same-depth only — no cross-section or reparenting.

- `PATCH /api/pages/reorder` bulk endpoint — transactional DB update
- Assigns gapped sort_order (10, 20, 30...) based on DOM order
- Vault frontmatter `order:` field updated after reorder
- Editor/admin role required
- Drop indicator (2px accent line) shows target position

### Version label in rail

Bottom of the app rail displays the current version (e.g. `v2.0.1`) fetched from `/api/version`. JetBrains Mono 10px, muted colour.

### Skill asset registration script

`scripts/register-skills.js` registers 50 assets (20 global skills, 5 agents, 14 superpowers, 9 job-app skills, 2 job-app agents) and 34 relationships into the KB Map view. Populates the relationship table and Mermaid diagram for navigating the Claude skill ecosystem.

## Rollback

```bash
# On NAS (via SSH):
DOCKER=/share/CACHEDEV2_DATA/.qpkg/container-station/usr/bin/docker

# Stop and remove current container
$DOCKER stop knowledge-base && $DOCKER rm knowledge-base

# Pull previous version
$DOCKER pull ghcr.io/sspaynter/knowledge-base:sha-88709ed

# Recreate with v2.0.0 image
$DOCKER run -d --name knowledge-base \
  --restart always \
  --network n8n-net \
  -p 32781:3000 \
  -v /share/CACHEDEV1_DATA/Container/knowledge-base/vault:/app/vault \
  -v /share/CACHEDEV1_DATA/Container/knowledge-base/data:/app/uploads \
  --env-file /share/Container/shared-secrets.env \
  -e NODE_ENV=production \
  -e PORT=3000 \
  -e DATABASE_URL=postgresql://kb_app:eb8aa0f9f18a0e279fc05f8b836cd328750d542acb3ba2f3@10.0.3.12:5432/nocodb \
  -e HQ_URL=https://hq.ss-42.com \
  -e VAULT_PATH=/app/vault \
  -e COOKIE_DOMAIN=.ss-42.com \
  ghcr.io/sspaynter/knowledge-base:sha-88709ed
```

Note: Migration 004 (archived status) is additive — rolling back the app does not require a DB rollback.
