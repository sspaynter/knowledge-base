# Knowledge Base v2.1.0

**Released:** 2026-03-05
**Tag:** [v2.1.0](https://github.com/sspaynter/knowledge-base/releases/tag/v2.1.0)
**Previous:** v2.0.1

## Summary

Replaces rsync-based vault sync with API-driven bidirectional sync. Claude sessions now push documentation to production immediately after writing and pull changes at session start. Server vault is authoritative — local deletes never propagate.

## Changes

### Added
- POST /api/pages/by-path — upsert pages from raw markdown with frontmatter
- GET /api/pages/export — bulk export with optional ?since= incremental filter
- kb-sync.sh — push a single vault file to the KB API
- kb-pull.sh — pull updated pages from the KB API to local vault
- LAN auto-detection in sync scripts
- Cloudflare Access bypass for /api/* paths
- chokidar suppressPath to prevent double-processing
- 15 new tests (134 total)

### Changed
- Service worker cache version bumped (kb-cache-v1 to kb-cache-v2)

### Security
- kb_app DB user: DELETE revoked on api_tokens and users
- Path traversal protection on by-path endpoint

### Infrastructure
- Vault directory ownership set to container user (UID 100)
- kb_app DB user granted full CRUD on knowledge_base schema

## Rollback

```bash
DOCKER=/share/CACHEDEV2_DATA/.qpkg/container-station/usr/bin/docker
$DOCKER pull ghcr.io/sspaynter/knowledge-base:v2.0.1
$DOCKER stop knowledge-base && $DOCKER rm knowledge-base
# Update docker-compose.yml image tag to :v2.0.1
$DOCKER compose up -d knowledge-base
```
